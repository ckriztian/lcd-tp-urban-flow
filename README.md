# Trabajo Práctico - Urban Flow

## Sprint actual
Sprint_1 - Inicialización, organización del entorno y preparación del análisis de datos.

## Objetivo
El objetivo del presente trabajo práctico es desarrollar un flujo de trabajo para el análisis de un conjunto de datos de infracciones de velocidad, aplicando tareas de organización, limpieza, procesamiento y documentación del proyecto.

## Introducción
En este trabajo se utiliza un dataset vinculado a multas por exceso de velocidad, compuesto por variables como patente, fecha, hora, velocidad registrada, velocidad máxima permitida, ubicación, identificador de radar y estado de la multa.

A partir de este conjunto de datos, se busca desarrollar un proceso ordenado de análisis que permita detectar inconsistencias, limpiar la información y preparar los datos para su posterior exploración e interpretación.

## Contexto del sprint de trabajo
Durante el Sprint_1 se abordaron las tareas iniciales necesarias para organizar el proyecto y dejar preparado el entorno de desarrollo. Entre las principales actividades realizadas se encuentran:

- creación y configuración del repositorio en GitHub
- definición de la estrategia de versionado mediante ramas
- implementación del flujo de trabajo colaborativo
- creación de la estructura de directorios del proyecto
- preparación del notebook principal según los criterios de evaluación
- documentación inicial del proyecto mediante los archivos `README.md` y `CHANGELOG.md`
